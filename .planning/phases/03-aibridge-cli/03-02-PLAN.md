    1. **src/busy_detector.rs** -- Create `BusyDetector` struct:
       - Fields (behind `Arc<Mutex<...>>` for shared access):
         - `idle: bool` (starts false)
         - `last_output: Instant` (last time output was seen)
         - `pattern: regex::Regex` (compiled busy pattern)
         - `idle_timeout: Duration` (default 500ms)
         - `tick_rate: Duration` (default 100ms)
       - Wrap mutable state in `Arc<tokio::sync::Mutex<BusyDetectorState>>` where state struct holds idle, last_output
       - `pub fn new(pattern: regex::Regex) -> Self`
       - `pub async fn process_line(&self, line: &str)`:
         - Lock state, set idle=false, update last_output to now
         - If line matches the busy pattern, this confirms busy state (already set idle=false)
       - `pub async fn is_idle(&self) -> bool` -- lock state, return idle flag
       - `pub async fn run(&self, on_idle: tokio::sync::mpsc::Sender<()>, cancel: tokio_util::sync::CancellationToken)`:
         - Spawn a tokio interval with tick_rate (100ms)
         - Each tick: lock state, if !idle && time since last_output > idle_timeout (500ms), set idle=true and send on on_idle channel
         - Stop when cancel is triggered
       - BRG-03: This is the core idle detection mechanism

    2. **src/queue.rs** -- Create `InjectionQueue` struct:
       - `const MAX_QUEUE_SIZE: usize = 100;`
       - Inner struct `Injection`:
         - `id: String` (uuid v4)
         - `text: String`
         - `priority: bool`
         - `sync_tx: Option<tokio::sync::oneshot::Sender<()>>` (None for async, Some for sync)
       - `InjectionQueue` wraps `Arc<tokio::sync::Mutex<Vec<Injection>>>`
       - `pub async fn enqueue(&self, text: String, priority: bool) -> Result<(String, usize), QueueError>`:
         - Lock, check len < MAX_QUEUE_SIZE or return Err(QueueError::Full)
         - Create Injection with uuid::Uuid::new_v4().to_string()
         - If priority: insert at index 0. Else: push to end
         - Return (id, position)
       - `pub async fn enqueue_sync(&self, text: String, priority: bool) -> Result<(String, tokio::sync::oneshot::Receiver<()>), QueueError>`:
         - Same as enqueue but creates oneshot channel, stores Sender in injection, returns Receiver
       - `pub async fn dequeue(&self) -> Option<Injection>`:
         - Lock, remove first item if any
       - `pub async fn clear(&self) -> usize`:
         - Lock, drain all items, return count removed
       - `pub async fn len(&self) -> usize`
       - `pub enum QueueError { Full }` with Display impl
       - BRG-05: FIFO queue with priority support, max 100 items

    1. **src/bridge.rs** -- Create `Bridge` struct:
       - Fields:
         - `pty: Arc<std::sync::Mutex<PtyHandler>>` (sync mutex since portable-pty is sync)
         - `detector: BusyDetector`
         - `queue: InjectionQueue`
         - `inject_notify: tokio::sync::mpsc::Sender<()>` (capacity 1 -- signal injection loop)
         - `cancel: tokio_util::sync::CancellationToken`
         - `paranoid: bool`
         - `child_running: Arc<std::sync::atomic::AtomicBool>`
         - `start_time: Instant` (for uptime in /status)
       - Add `tokio-util` to Cargo.toml dependencies: `tokio-util = { version = "0.7", features = ["rt"] }`

       - `pub fn new(command: &[String], pattern: regex::Regex, paranoid: bool, inject_delay_ms: u64) -> Result<Self>`:
         - Get terminal size via terminal_size crate (add `terminal-size = "0.4"` to Cargo.toml) or default (80, 24)
         - Spawn PtyHandler with command and size
         - Create BusyDetector with pattern
         - Create InjectionQueue
         - Create mpsc channel (capacity 1) for inject_notify
         - Create CancellationToken

       - `pub async fn run(&self) -> Result<()>`:
         - Spawns 4 concurrent tokio tasks (mirrors Go reference's 5 goroutines, minus HTTP which is separate):

         **Task 1 -- PTY Read Loop** (spawn_blocking since PTY is sync):
         - Loop: read 4KB from pty.read(), write to stdout
         - For each chunk: call ansi::strip_ansi, split by newlines, call detector.process_line for each line
         - On read error or child exit: set child_running=false, cancel token, break

         **Task 2 -- Stdin Forward** (spawn_blocking since stdin is sync):
         - Loop: read from stdin (raw mode), write to pty.write_all()
         - Coordinate with injection via a write lock (Arc<std::sync::Mutex<>> around PTY writer)
         - On error: break

         **Task 3 -- Detector Ticker**:
         - Call detector.run(on_idle_tx, cancel_token.clone())
         - When idle detected, also send on inject_notify to wake injection loop

         **Task 4 -- Injection Loop**:
         - Wait on inject_notify receiver
         - When signaled: while detector.is_idle(), dequeue from queue
         - For each item: call pty.inject_text(item.text)
         - If item has sync_tx: send completion signal via oneshot
         - If queue empty or detector not idle: go back to waiting

       - `pub fn notify_enqueue(&self)`:
         - Non-blocking send on inject_notify (try_send, ignore if already signaled)

       - `pub async fn is_idle(&self) -> bool` -- delegate to detector
       - `pub async fn queue_len(&self) -> usize` -- delegate to queue
       - `pub fn is_child_running(&self) -> bool` -- read AtomicBool
       - `pub fn uptime(&self) -> Duration` -- Instant::now() - start_time
       - `pub fn queue(&self) -> &InjectionQueue` -- getter

    2. **src/main.rs** -- Add `mod bridge; mod busy_detector; mod queue;` declarations. No functional changes yet (wiring in Plan 04).

    3. **Terminal raw mode**: In the bridge.run() method, before starting tasks:
       - Save current terminal state using `crossterm` or manual libc calls
       - Set stdin to raw mode so keypresses are forwarded immediately
       - On shutdown (cancel token triggered): restore terminal state
       - Add `crossterm = "0.28"` to Cargo.toml for raw mode management
       - Use defer-like pattern: store old state, restore in a drop guard or on cancel
