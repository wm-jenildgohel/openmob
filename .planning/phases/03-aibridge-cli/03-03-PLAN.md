  <action>
    1. **src/handlers.rs** -- Create all endpoint handlers. Shared state: `Arc<Bridge>` passed via axum State extractor.

       **Request/Response types** (serde Serialize/Deserialize):
       ```
       InjectRequest { text: String, priority: Option<bool> }
       InjectResponse { id: String, queued: bool, position: usize }
       HealthResponse { status: String, version: String }
       StatusResponse { idle: bool, queue_length: usize, child_running: bool, uptime_seconds: u64 }
       QueueClearResponse { cleared: usize }
       ErrorResponse { error: String }
       ```

       **GET /health** (`pub async fn handle_health`):
       - Return 200 with `{ "status": "ok", "version": "0.1.0" }` (read from env!("CARGO_PKG_VERSION"))
       - BRG-06

       **GET /status** (`pub async fn handle_status`):
       - Call bridge.is_idle(), bridge.queue_len(), bridge.is_child_running(), bridge.uptime()
       - Return 200 with StatusResponse
       - BRG-06

       **POST /inject** (`pub async fn handle_inject`):
       - Parse JSON body as InjectRequest. Return 400 if invalid or text is empty.
       - If !bridge.is_child_running(): return 503 "child process not running"
       - Check query param `sync` (axum Query extractor): if sync=true, use sync path
       - **Async path**: call queue.enqueue(text, priority.unwrap_or(false)), bridge.notify_enqueue(), return 200 with InjectResponse { queued: true, position }
       - **Sync path** (BRG-09): call queue.enqueue_sync(text, priority), bridge.notify_enqueue(). Then tokio::select! on oneshot receiver vs tokio::time::sleep(timeout). If receiver fires: return 200 with InjectResponse { queued: false, position: 0 }. If timeout: return 408 "injection timeout".
       - Queue full: return 429 "queue full (max 100)"
       - BRG-02, BRG-09

       **DELETE /queue** (`pub async fn handle_queue_clear`):
       - Call queue.clear(), return 200 with QueueClearResponse { cleared: count }

    2. **src/server.rs** -- Create server module:
       - `pub struct AppState { pub bridge: Arc<Bridge>, pub timeout: Duration }`
       - `pub fn create_router(state: AppState) -> axum::Router`:
         - `axum::Router::new()`
         - `.route("/health", get(handle_health))`
         - `.route("/status", get(handle_status))`
         - `.route("/inject", post(handle_inject))`
         - `.route("/queue", delete(handle_queue_clear))`
         - `.with_state(Arc::new(state))`
       - `pub async fn start_server(host: &str, port: u16, router: axum::Router) -> Result<()>`:
         - Parse addr as `format!("{}:{}", host, port)` -- BRG-10: defaults to 127.0.0.1
         - `let listener = tokio::net::TcpListener::bind(&addr).await?;`
         - `axum::serve(listener, router).await?;`
         - Log "AiBridge HTTP server listening on {addr}" to stderr

    3. **src/main.rs** -- Add `mod server; mod handlers;` declarations. Still no full wiring (Plan 04 does that).

    4. **CORS middleware** (optional but useful for browser-based tools):
       - Add `tower-http = { version = "0.6", features = ["cors"] }` to Cargo.toml
       - Apply CorsLayer::permissive() to router (localhost-only server, permissive CORS is safe)
