  <action>
    Update cmd/aibridge/main.go to wire everything together:

    1. Add tool detection function checkRequiredTools():
       - Check if the wrapped command exists in PATH using exec.LookPath
       - If the command is not found, print helpful message:
         - "Error: '{command}' not found in PATH"
         - "Make sure {command} is installed and available in your PATH"
       - Return error if command not found

    2. Add a "version" subcommand using cobra (in addition to --version flag):
       - `aibridge version` prints version, Go version, OS/arch

    3. Update the run function (rootCmd.RunE) to:
       a. If flagVersion: print version and return nil

       b. Configure slog: if flagVerbose, set level to Debug; else Info. Use slog.SetDefault with text handler.

       c. Get command args: args after -- (cobra.ArbitraryArgs handles this, or use cmd.Flags().Args())

       d. Call checkRequiredTools() for the command -- exit with instructions if missing

       e. Build BridgeConfig from flags:
          ```go
          cfg := bridge.BridgeConfig{
              Command:     args,
              Port:        flagPort,
              Host:        flagHost,
              BusyPattern: flagBusyPattern,
              Timeout:     flagTimeout,
              Verbose:     flagVerbose,
              Paranoid:    flagParanoid,
              InjectDelay: flagInjectDelay,
          }
          ```

       f. Create bridge: b := bridge.NewBridge(cfg)

       g. Create server: s := server.NewServer(flagHost, flagPort, b, time.Duration(flagTimeout)*time.Second)

       h. Set up signal handler:
          - sigCh := make(chan os.Signal, 1)
          - signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
          - Goroutine: wait for signal -> slog.Info("shutting down") -> b.Stop() -> s.Shutdown(5s context) -> os.Exit(0)

       i. Start bridge: if err := b.Start(); err != nil -> fatal

       j. Start HTTP server in goroutine: go func() { if err := s.Start(); err != nil -> log if not context canceled }()

       k. Log startup info: "AiBridge v{version} wrapping '{command}' on {host}:{port}"
          - If paranoid: log "paranoid mode: text injected without auto-submit"

       l. Wait for bridge child to exit: b.Wait() or context done

       m. Clean up: b.Stop(), s.Shutdown(5s)

    4. Add recover() in main() to ensure terminal state is restored on panic:
       ```go
       func main() {
           defer func() {
               if r := recover(); r != nil {
                   fmt.Fprintf(os.Stderr, "panic: %v\n", r)
                   os.Exit(1)
               }
           }()
           if err := rootCmd.Execute(); err != nil {
               os.Exit(1)
           }
       }
       ```

    5. Ensure "wrap" is supported as alias -- either as a Cobra subcommand that delegates to root, or by accepting `aibridge wrap -- claude` syntax. Simplest: make the root command accept the command directly after flags: `aibridge --port 9999 -- claude` AND add a "wrap" cobra subcommand that does the same thing.

  <action>
    Create openmob_bridge/Makefile:

    1. Variables:
       - BINARY_NAME := aibridge
       - VERSION := 0.1.0
       - BUILD_DIR := build
       - GO := go (or override with GO env var)
       - LDFLAGS := -s -w (strip debug info for smaller binaries)
       - CGO_ENABLED := 0 (fully static binaries)

    2. Targets:

       - `build`: Build for current platform
         ```
         $(GO) build -ldflags "$(LDFLAGS)" -o $(BUILD_DIR)/$(BINARY_NAME) ./cmd/aibridge
         ```

       - `build-all`: Cross-compile for all platforms
         - linux/amd64: GOOS=linux GOARCH=amd64
         - linux/arm64: GOOS=linux GOARCH=arm64
         - darwin/amd64: GOOS=darwin GOARCH=amd64
         - darwin/arm64: GOOS=darwin GOARCH=arm64
         - windows/amd64: GOOS=windows GOARCH=amd64 (output: aibridge.exe)
         Each produces: $(BUILD_DIR)/$(BINARY_NAME)-$(GOOS)-$(GOARCH)[.exe]

       - `install`: Build and copy to $GOPATH/bin or /usr/local/bin
         ```
         $(GO) install ./cmd/aibridge
         ```

       - `clean`: Remove build directory
         ```
         rm -rf $(BUILD_DIR)
         ```

       - `check-go`: Verify Go is installed, print helpful message if not
         ```
         @which $(GO) > /dev/null 2>&1 || (echo ""; \
           echo "ERROR: Go is not installed or not in PATH"; \
           echo ""; \
           echo "Install Go:"; \
           echo "  macOS:   brew install go"; \
           echo "  Ubuntu:  sudo snap install go --classic"; \
           echo "  Fedora:  sudo dnf install golang"; \
           echo "  Windows: Download from https://go.dev/dl/"; \
           echo "  Manual:  wget https://go.dev/dl/go1.26.1.linux-amd64.tar.gz && sudo tar -C /usr/local -xzf go1.26.1.linux-amd64.tar.gz"; \
           echo ""; \
           echo "Then add to PATH: export PATH=\$$PATH:/usr/local/go/bin"; \
           echo ""; \
           exit 1)
         ```

       - `deps`: Download Go module dependencies
         ```
         $(GO) mod download
         ```

       - `fmt`: Format code
         ```
         $(GO) fmt ./...
         ```

       - `vet`: Run go vet
         ```
         $(GO) vet ./...
         ```

       - `release`: Create tarball/zip archives for each platform
         For each platform binary in build/:
         - Linux/macOS: tar -czf $(BUILD_DIR)/$(BINARY_NAME)-$(VERSION)-$(GOOS)-$(GOARCH).tar.gz
         - Windows: zip $(BUILD_DIR)/$(BINARY_NAME)-$(VERSION)-windows-amd64.zip

       - `.PHONY`: List all targets

    3. Make `build` depend on `check-go`
    4. Make `build-all` depend on `check-go` and `clean`

    The Makefile should be self-documenting: `make help` target that greps for ## comments.
