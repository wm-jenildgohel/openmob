  </read_first>
  <action>
    Rewrite `main.rs` to wire all components together. The main function flow:

    1. **Parse CLI args** via clap derive (already scaffolded). Extract: command, port, host, busy_pattern, timeout, verbose, paranoid, inject_delay.

    2. **Validate command** -- if command vec is empty, print error and exit. The command[0] is the program to wrap, command[1..] are its args.

    3. **Tool detection at startup**:
       - Use `which::which(command[0])` to check if the tool exists in PATH
       - Add `which = "7"` to Cargo.toml
       - If tool not found: print clear error message to stderr:
         ```
         Error: '{}' not found in PATH.

         Install guides:
           claude  -> npm install -g @anthropic-ai/claude-code
           codex   -> npm install -g @openai/codex
           gemini  -> npm install -g @anthropic-ai/gemini-cli
           other   -> ensure the command is installed and in your PATH
         ```
       - Exit with code 1

    4. **Resolve agent pattern**:
       - Call `patterns::resolve_pattern(&command[0], cli.busy_pattern.as_deref())`
       - If verbose: log detected agent name and pattern to stderr

    5. **Create Bridge**:
       - `let bridge = Arc::new(Bridge::new(&cli.command, pattern, cli.paranoid, cli.inject_delay)?);`

    6. **Create HTTP server**:
       - `let state = AppState { bridge: bridge.clone(), timeout: Duration::from_secs(cli.timeout) };`
       - `let router = server::create_router(state);`

    7. **Signal handling** (SIGINT, SIGTERM):
       - Use `tokio::signal::ctrl_c()` for cross-platform Ctrl+C
       - On Unix: also handle SIGTERM via `tokio::signal::unix::signal(SignalKind::terminate())`
       - On signal: trigger bridge cancel token, restore terminal, exit

    8. **Start everything concurrently** with tokio::select!:
       ```rust
       tokio::select! {
           result = bridge.run() => { /* PTY exited (child process done) */ }
           result = server::start_server(&cli.host, cli.port, router) => { /* server error */ }
           _ = tokio::signal::ctrl_c() => { /* user interrupted */ }
       }
       ```
       - After select: ensure terminal is restored, bridge is shut down, child is killed
       - Print exit message to stderr

    9. **Startup banner** (to stderr, not stdout -- stdout is the PTY):
       ```
       AiBridge v{version}
       Wrapping: {command}
       Agent: {detected_agent_name or "custom"}
       HTTP API: http://{host}:{port}
       Paranoid: {yes/no}
       ```

    10. **Terminal restore guard**: Create a struct that implements Drop to restore terminal state even on panic. This is critical -- if terminal is left in raw mode after crash, user has broken shell.

    1. **Makefile** -- Create with targets for building and cross-compiling:

       ```makefile
       BINARY_NAME := aibridge
       VERSION := $(shell cargo metadata --format-version=1 --no-deps | python3 -c "import sys,json; print(json.load(sys.stdin)['packages'][0]['version'])" 2>/dev/null || echo "0.1.0")

       # Default: build debug
       .PHONY: build
       build:
       	cargo build

       # Release build for current platform
       .PHONY: release
       release:
       	cargo build --release

       # Install to ~/.cargo/bin
       .PHONY: install
       install:
       	cargo install --path .

       # Cross-compile all targets (requires `cross` tool: cargo install cross)
       .PHONY: cross-all
       cross-all: cross-linux-amd64 cross-linux-arm64 cross-darwin-amd64 cross-darwin-arm64 cross-windows-amd64

       .PHONY: cross-linux-amd64
       cross-linux-amd64:
       	cross build --release --target x86_64-unknown-linux-gnu
       	mkdir -p dist
       	cp target/x86_64-unknown-linux-gnu/release/$(BINARY_NAME) dist/$(BINARY_NAME)-linux-amd64

       .PHONY: cross-linux-arm64
       cross-linux-arm64:
       	cross build --release --target aarch64-unknown-linux-gnu
       	mkdir -p dist
       	cp target/aarch64-unknown-linux-gnu/release/$(BINARY_NAME) dist/$(BINARY_NAME)-linux-arm64

       .PHONY: cross-darwin-amd64
       cross-darwin-amd64:
       	cross build --release --target x86_64-apple-darwin
       	mkdir -p dist
       	cp target/x86_64-apple-darwin/release/$(BINARY_NAME) dist/$(BINARY_NAME)-darwin-amd64

       .PHONY: cross-darwin-arm64
       cross-darwin-arm64:
       	cross build --release --target aarch64-apple-darwin
       	mkdir -p dist
       	cp target/aarch64-apple-darwin/release/$(BINARY_NAME) dist/$(BINARY_NAME)-darwin-arm64

       .PHONY: cross-windows-amd64
       cross-windows-amd64:
       	cross build --release --target x86_64-pc-windows-gnu
       	mkdir -p dist
       	cp target/x86_64-pc-windows-gnu/release/$(BINARY_NAME).exe dist/$(BINARY_NAME)-windows-amd64.exe

       .PHONY: clean
       clean:
       	cargo clean
       	rm -rf dist

       .PHONY: check
       check:
       	cargo check

       .PHONY: fmt
       fmt:
       	cargo fmt
       ```
