  <action>
    1. Create `openmob_bridge/` directory and `cargo init` (or write Cargo.toml manually).

    2. **Cargo.toml** -- Set package name to `aibridge`, edition 2021. Dependencies:
       - `portable-pty = "0.8"` (PTY management, cross-platform Unix+Windows)
       - `clap = { version = "4", features = ["derive"] }` (CLI parsing)
       - `axum = "0.8"` (HTTP server -- used in Plan 03 but declare now)
       - `tokio = { version = "1", features = ["full"] }` (async runtime)
       - `strip-ansi-escapes = "0.2"` (ANSI code removal)
       - `regex = "1"` (idle detection patterns)
       - `serde = { version = "1", features = ["derive"] }` (serialization)
       - `serde_json = "1"` (JSON)
       - `uuid = { version = "1", features = ["v4"] }` (injection IDs)

    3. **src/ansi.rs** -- Create `pub fn strip_ansi(input: &[u8]) -> String` that:
       - Uses `strip-ansi-escapes::strip` to remove all ANSI escape sequences
       - Returns a clean UTF-8 string (lossy conversion for non-UTF8 bytes)
       - This is used by the BusyDetector in Plan 02 before regex matching

    4. **src/patterns.rs** -- Create:
       - `pub struct AgentPattern { pub name: &'static str, pub busy_regex: &'static str }`
       - `pub fn builtin_patterns() -> Vec<AgentPattern>` returning patterns for:
         - claude: busy_regex = `(?i)thinking` (Claude Code shows "thinking" while processing)
         - codex: busy_regex = `(?i)esc to interrupt` (Codex shows this while running)
         - gemini: busy_regex = `(?i)esc to cancel` (Gemini shows this while running)
       - `pub fn detect_agent(command: &str) -> Option<&AgentPattern>` -- matches command string against known agent names
       - `pub fn resolve_pattern(command: &str, custom_pattern: Option<&str>) -> regex::Regex`:
         - If custom_pattern provided (from --busy-pattern flag), compile and return it (BRG-08)
         - Otherwise auto-detect from command name via detect_agent
         - Fallback to default pattern `(?i)esc to interrupt` if no match

    5. **src/pty_handler.rs** -- Create `PtyHandler` struct:
       - Fields: `child: Box<dyn portable_pty::Child + Send>`, `master: Box<dyn portable_pty::MasterPty + Send>`, `reader: Box<dyn std::io::Read + Send>`, `writer: Box<dyn std::io::Write + Send>`, `paranoid: bool`, `inject_delay_ms: u64`
       - `pub fn spawn(command: &[String], size: (u16, u16)) -> Result<PtyHandler>`:
         - Create `portable_pty::native_pty_system()`
         - Configure PtySize from terminal size (or default 80x24)
         - Open PTY pair, spawn command as child process
         - Get reader/writer pair from master
       - `pub fn read(&mut self, buf: &mut [u8]) -> std::io::Result<usize>` -- reads from PTY master
       - `pub fn write_all(&mut self, data: &[u8]) -> std::io::Result<()>` -- writes to PTY master (user input forwarding)
       - `pub fn inject_text(&mut self, text: &str) -> std::io::Result<()>`:
         - Write text bytes to PTY master
         - If not paranoid: sleep inject_delay_ms, then write `\r` (Enter) (BRG-07)
       - `pub fn resize(&self, cols: u16, rows: u16) -> Result<()>` -- resize PTY
       - `pub fn is_alive(&mut self) -> bool` -- check if child process is still running
       - `pub fn kill(&mut self)` -- kill child process
       - Note: All PTY I/O is synchronous (portable-pty is sync). The bridge in Plan 02 will use tokio::task::spawn_blocking for the read loop.

    6. **src/main.rs** -- Create clap CLI skeleton:
       - `#[derive(Parser)] struct Cli` with fields:
         - `command: Vec<String>` (positional, the wrapped command + args)
         - `#[arg(short, long, default_value = "9999")] port: u16`
         - `#[arg(long, default_value = "127.0.0.1")] host: String`
         - `#[arg(long)] busy_pattern: Option<String>` (BRG-08)
         - `#[arg(short, long, default_value = "300")] timeout: u64` (seconds, BRG-09)
         - `#[arg(short, long)] verbose: bool`
         - `#[arg(long)] paranoid: bool` (BRG-07)
         - `#[arg(long, default_value = "50")] inject_delay: u64` (ms between text and Enter)
       - `mod pty_handler; mod ansi; mod patterns;` declarations
       - `#[tokio::main] async fn main()` that parses CLI args and prints them (full wiring in Plan 04)
       - Add version string: `const VERSION: &str = env!("CARGO_PKG_VERSION");`
  </action>
