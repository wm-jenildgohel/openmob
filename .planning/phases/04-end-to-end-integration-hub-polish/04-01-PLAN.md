**openmob_hub/lib/services/process_manager.dart:**
- Import `dart:io`, `dart:async`, `dart:convert`, `package:rxdart/rxdart.dart`, `package:http/http.dart` as http
- Import `../models/process_info.dart`, `log_service.dart`
- Class `ProcessManager` with `LogService _logService` constructor param

**State management:**
- `final _mcpStatus = BehaviorSubject<ProcessInfo>.seeded(ProcessInfo(name: 'MCP Server', status: ProcessStatus.stopped))`
- `final _bridgeStatus = BehaviorSubject<ProcessInfo>.seeded(ProcessInfo(name: 'AiBridge', status: ProcessStatus.stopped))`
- `ValueStream<ProcessInfo> get mcpStatus$` and `ValueStream<ProcessInfo> get bridgeStatus$` getters
- `Process? _mcpProcess`
- `Timer? _bridgeHealthTimer`

**Path resolution:**
- `String get _projectRoot` -- walk up from `Directory.current.path` to find a directory that contains `openmob_mcp/` and `openmob_bridge/` subdirectories. Fallback: use `Directory.current.parent.path`
- `String get _mcpDir` => `'$_projectRoot/openmob_mcp'`
- `String get _aibridgePath` => `'$_projectRoot/openmob_bridge/target/release/aibridge'`

**startMcp() async:**
1. Guard: if `_mcpStatus.value.status == ProcessStatus.running`, return
2. `_mcpStatus.add(_mcpStatus.value.copyWith(status: ProcessStatus.starting))`
3. Try: `_mcpProcess = await Process.start('node', ['build/app/index.js'], workingDirectory: _mcpDir)`
4. Listen `_mcpProcess!.stdout.transform(utf8.decoder).transform(const LineSplitter())` -- forward each line to `_logService.addLine('mcp', line)`
5. Same for stderr: `_logService.addLine('mcp', line, level: LogLevel.warning)` (MCP uses stderr for status messages)
6. `_mcpStatus.add(ProcessInfo(name: 'MCP Server', status: ProcessStatus.running, pid: _mcpProcess!.pid, startedAt: DateTime.now()))`
7. `_logService.addLine('hub', 'MCP Server started (PID: ${_mcpProcess!.pid})')`
8. Listen for exit: `_mcpProcess!.exitCode.then((code) { ... })` -- on exit, set status to stopped (or error if code != 0), log the event, set `_mcpProcess = null`
9. Catch: set status to error with errorMessage, log error

**stopMcp() async:**
1. If `_mcpProcess != null`: `_mcpProcess!.kill()`, wait up to 3 seconds, force kill if needed
2. `_mcpStatus.add(ProcessInfo(name: 'MCP Server', status: ProcessStatus.stopped))`
3. `_mcpProcess = null`
4. `_logService.addLine('hub', 'MCP Server stopped')`

**restartMcp() async:**
1. `await stopMcp()`
2. `await Future.delayed(const Duration(seconds: 1))`
3. `await startMcp()`

**AiBridge health polling (HUB-05):**
AiBridge needs its own PTY terminal so the Hub cannot directly start it. Instead, monitor via health polling.
- `void startBridgeMonitoring()` -- creates `_bridgeHealthTimer = Timer.periodic(const Duration(seconds: 3), (_) => _pollBridgeHealth())`
- `Future<void> _pollBridgeHealth()` async:
  - Try: `final response = await http.get(Uri.parse('http://127.0.0.1:9999/health')).timeout(const Duration(seconds: 2))`
  - If 200: set bridgeStatus to running (if wasn't running before, log 'AiBridge detected')
  - On any error (SocketException, TimeoutException): set bridgeStatus to stopped (if was running before, log 'AiBridge disconnected')
- Call `startBridgeMonitoring()` in constructor so polling starts immediately

**dispose():**
1. `_mcpProcess?.kill()` -- clean up child process
2. `_bridgeHealthTimer?.cancel()`
3. `_mcpStatus.close()`
4. `_bridgeStatus.close()`
