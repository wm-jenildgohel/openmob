**startMcp() async method:**
1. If already running, return
2. Update status to `starting`
3. Check if `node` is in PATH (which already done by SystemCheck, but guard here too)
4. Run `Process.start('node', ['build/app/index.js'], workingDirectory: mcpDir)` with `ProcessStartMode.normal`
5. Listen to stdout and stderr lines, forward each to `logService.addLine('mcp', line)`
6. Update status to `running` with pid
7. Listen for process exit, update status to `stopped` (or `error` if non-zero exit code)
8. Log start/stop events

**stopMcp() async method:**
1. Kill process via `_mcpProcess?.kill()`
2. Update status to `stopped`
3. Log stop event

**startBridge(String agentCommand) async method:**
1. Similar pattern to startMcp
2. Run: `Process.start(aibridgePath, ['--', agentCommand])` -- NOTE: AiBridge needs a PTY so this is a simplified "launch for monitoring" mode. The Hub starts aibridge as a subprocess for lifecycle tracking. The user interacts with it via their own terminal.
3. Actually -- rethink: the Hub should NOT wrap AiBridge in a subprocess since AiBridge needs a real PTY (terminal). Instead:
   - `startBridge` should launch aibridge process detached
   - Monitor by polling `http://127.0.0.1:9999/health` every 3 seconds
   - If health check fails 3 times in a row, mark as stopped
4. Use `Process.start(aibridgePath, args, mode: ProcessStartMode.detached)` for AiBridge
5. Actually simpler: For AiBridge, since it needs its own terminal, ProcessManager should just track status by health-polling `GET http://127.0.0.1:9999/health`. The user starts aibridge themselves in a separate terminal.
6. Final design for AiBridge tracking:
   - `_bridgeHealthTimer`: periodic Timer that GETs `http://127.0.0.1:9999/health` every 3s
   - If response 200 -> status = running
   - If connection refused -> status = stopped
   - Start timer in constructor, dispose in dispose()

**stopBridge() async method:**
1. Cannot directly stop external aibridge -- just update UI status to reflect health poll results
2. Log that user should Ctrl+C in the AiBridge terminal

**restartMcp() async:**
1. await stopMcp()
2. await Future.delayed(Duration(seconds: 1))
3. await startMcp()

**dispose():**
1. Kill MCP process if running
2. Cancel bridge health timer
3. Close both BehaviorSubjects
