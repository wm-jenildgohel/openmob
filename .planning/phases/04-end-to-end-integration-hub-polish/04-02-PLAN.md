**openmob_hub/lib/ui/widgets/live_preview.dart:**
- Controller + Widget pattern to avoid setState:
- Class `LivePreviewController` (NOT a widget):
  - Constructor: `String deviceId`
  - `final _image = BehaviorSubject<Uint8List?>.seeded(null)`
  - `final _loading = BehaviorSubject<bool>.seeded(false)`
  - `ValueStream<Uint8List?> get image$`
  - `ValueStream<bool> get loading$`
  - `Timer? _timer`
  - `void start()` -- `_timer = Timer.periodic(Duration(seconds: 2), (_) => _fetch())`
  - `Future<void> _fetch()` async:
    - If `_loading.value` return (skip if still fetching)
    - `_loading.add(true)`
    - Try: call `screenshotService.captureScreenshot(deviceId)`, decode `base64Decode(result.base64)` to Uint8List, `_image.add(bytes)`
    - Catch: silently ignore (device might be disconnected)
    - Finally: `_loading.add(false)`
  - `void stop()` -- `_timer?.cancel(); _timer = null`
  - `void dispose()` -- stop(), _image.close(), _loading.close()
- Widget `LivePreview` (StatelessWidget):
  - Takes `LivePreviewController controller`
  - `ValueStreamBuilder<Uint8List?>` on `controller.image$`
  - null state: Container with ResColors.surface bg, Icons.phone_android icon, "Waiting for screenshot..." text, centered
  - bytes state: `Image.memory(bytes!, fit: BoxFit.contain, gaplessPlayback: true)` -- gaplessPlayback:true prevents flicker on update
  - Wrap in ClipRRect with borderRadius 8 and Container with ResColors.surface background
