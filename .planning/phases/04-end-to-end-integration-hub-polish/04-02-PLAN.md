**openmob_hub/lib/ui/widgets/live_preview.dart:**
- StatefulWidget `LivePreview` with `String deviceId` param -- EXCEPTION: needs StatefulWidget for Timer lifecycle, but all STATE is in BehaviorSubject
- Actually: make it a StatelessWidget that takes a `Stream<Uint8List?>` or simpler: use a dedicated BehaviorSubject internally
- Better approach: StatelessWidget wrapper, internal logic uses a `BehaviorSubject<Uint8List?>` for the image bytes
- Final design: Class `LivePreviewController` (not a widget) that holds `BehaviorSubject<Uint8List?>` and a `Timer`. Widget `LivePreview` takes the controller.
- `LivePreviewController`:
  - Constructor takes `String deviceId`, `ScreenshotService screenshotService`
  - `final _image = BehaviorSubject<Uint8List?>.seeded(null)`
  - `ValueStream<Uint8List?> get image$`
  - `Timer? _timer`
  - `void start()` -- creates Timer.periodic(Duration(seconds: 2)) that calls `_fetchScreenshot()`
  - `_fetchScreenshot()` async: calls `screenshotService.captureScreenshot(deviceId)`, decodes base64 to Uint8List, pushes to `_image`
  - `void stop()` -- cancels timer
  - `void dispose()` -- stop + close subject
- `LivePreview` widget (StatelessWidget):
  - Takes `LivePreviewController controller`
  - Uses `ValueStreamBuilder<Uint8List?>` on `controller.image$`
  - If null: show placeholder with device icon and "Waiting for screenshot..." text
  - If bytes: `Image.memory(bytes, fit: BoxFit.contain)` inside a `Container` with `ResColors.surface` background and rounded corners
  - Aspect ratio container to maintain device proportions
