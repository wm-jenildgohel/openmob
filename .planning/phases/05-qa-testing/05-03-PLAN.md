Create the Testing screen as a StatelessWidget (all state via rxdart BehaviorSubject, NEVER setState). Use ResColors throughout.

**Layout structure (Column):**

Top section: Header row with "Testing" title + action buttons:
- "New Script" button (opens a dialog to create test script)
- "Run Flutter Test" button (opens dialog for flutter test path input)

Middle section: Two-column Row (Expanded each):

**Left column -- Script List + Editor:**
- ValueStreamBuilder on testRunnerService.scripts$ to show list of saved scripts
- Each script shown as a Card with: name, device ID, step count, created date
- Card has trailing IconButtons: play (run), delete
- When a script card is tapped, show its JSON in a read-only Text area below the list
- "New Script" dialog: TextField for name, TextField for deviceId (or dropdown from deviceManager.devices$), a multiline TextField for JSON steps input (user types JSON array of steps), Save button that parses JSON, creates TestScript, calls testRunnerService.addScript

Use a module-level BehaviorSubject for selected script: `final _selectedScript = BehaviorSubject<String?>.seeded(null);`

**Right column -- Results Dashboard:**
- ValueStreamBuilder on testRunnerService.currentRun$ for active test indicator (show a LinearProgressIndicator with ResColors.testRunning when non-null)
- ValueStreamBuilder on testRunnerService.results$ for results list
- Summary row at top: total tests, passed count (green chip), failed count (red chip)
- Results list: each TestResult as an ExpansionTile:
  - Title: script name + status icon (checkmark green for passed, X red for failed, spinner for running)
  - Subtitle: "X/Y steps passed | Xms" with timing
  - Children (expanded): List of StepResult entries, each showing:
    - Step index + action name
    - Pass/fail icon (ResColors.testPassed / ResColors.testFailed)
    - Duration in ms
    - If error: Text with error message in ResColors.testFailed color
    - If screenshotBase64 is non-null: show "View Screenshot" button that opens a Dialog with Image.memory(base64Decode(screenshotBase64)) -- import dart:convert for base64Decode, dart:typed_data for Uint8List

**"Run Flutter Test" dialog:**
- TextField for test file path (e.g. "test/widget_test.dart" or "integration_test/app_test.dart")
- Optional device selector (dropdown from deviceManager.devices$)
- Run button: creates TestScript with flutterTestPath set, adds it, runs it

All colors use ResColors. All stream subscriptions use ValueStreamBuilder. No setState anywhere.
