**openmob_hub/lib/models/test_script.dart:**
- `TestStep` class with fields: `action` (String: tap, swipe, type_text, press_key, launch_app, terminate_app, open_url, go_home, gesture, wait), `params` (Map&lt;String, dynamic&gt; -- x, y, index, text, package, url, duration, etc.), `assertion` (optional Map&lt;String, dynamic&gt; with `type` key: "element_exists", "element_text", "screenshot_match", "none"), `description` (optional String for display)
- `TestScript` class with fields: `id` (String, auto-generated UUID via DateTime.now().millisecondsSinceEpoch.toString()), `name` (String), `deviceId` (String), `steps` (List&lt;TestStep&gt;), `createdAt` (DateTime), `flutterTestPath` (optional String -- when set, run flutter test/drive instead of steps)
- Both classes: `toJson()`, `fromJson()` factory constructors
- Steps support JSON format: `{"action": "tap", "params": {"x": 100, "y": 200}, "assertion": {"type": "element_exists", "index": 3}}`

**openmob_hub/lib/models/test_result.dart:**
- `StepResult` class: `stepIndex` (int), `action` (String), `success` (bool), `error` (String?), `durationMs` (int), `screenshotBase64` (String? -- captured on failure), `assertionResult` (Map&lt;String, dynamic&gt;?)
- `TestResult` class: `scriptId` (String), `scriptName` (String), `status` (TestStatus enum: passed, failed, running, error), `steps` (List&lt;StepResult&gt;), `totalDurationMs` (int), `passedCount` (int getter), `failedCount` (int getter), `startedAt` (DateTime), `completedAt` (DateTime?)
- Both classes: `toJson()`, `fromJson()` factory constructors

**openmob_hub/lib/services/test_runner_service.dart:**
- `TestRunnerService` class with constructor DI: `ActionService`, `ScreenshotService`, `DeviceManager`, `LogService`
- BehaviorSubject fields: `_scripts` (BehaviorSubject&lt;List&lt;TestScript&gt;&gt;), `_results` (BehaviorSubject&lt;List&lt;TestResult&gt;&gt;), `_currentRun` (BehaviorSubject&lt;TestResult?&gt;)
- Expose: `scripts$`, `results$`, `currentRun$` as ValueStream getters
- `addScript(TestScript)` -- adds to _scripts list
- `removeScript(String id)` -- removes by id
- `getScript(String id)` -- returns TestScript? from _scripts.value
- `runScript(String scriptId)` -- main execution method:
  - Looks up script by id, validates device exists via DeviceManager
  - If script.flutterTestPath is set, delegate to `_runFlutterTest()`
  - Otherwise iterate steps: for each TestStep, call the corresponding ActionService method based on step.action (use a switch on action string: "tap" calls actionService.tap with params x,y or tapElement with params index, "swipe" calls swipe with x1,y1,x2,y2,duration, "type_text" calls typeText, "press_key" calls pressKey with keyCode, "launch_app" calls launchApp, "terminate_app" calls terminateApp, "open_url" calls openUrl, "go_home" calls goHome, "gesture" calls gesture, "wait" does Future.delayed with params duration)
  - After each action, if step has assertion: execute assertion (for "element_exists": call dm.getDevice to check, use UiTreeService -- but keep simple: for now assertion checks are logged, assertion type "screenshot_match" captures a screenshot via screenshotService). Mark step pass/fail.
  - On failure: capture screenshot via screenshotService.captureScreenshot, store base64 in StepResult.screenshotBase64
  - Track timing per step (Stopwatch)
  - Build StepResult for each step, accumulate into TestResult
  - Update _currentRun during execution, add to _results when complete
  - Log progress via logService.addLine('test', ...)
- `_runFlutterTest(TestScript script)` -- runs flutter test as child process:
  - Resolve project root using walk-up pattern (same as ProcessManager: check for pubspec.yaml up to 5 levels)
  - If flutterTestPath ends with _test.dart, run: `Process.start('flutter', ['test', script.flutterTestPath])` in project root
  - If flutterTestPath contains 'drive' or 'integration', run: `Process.start('flutter', ['drive', '--target', script.flutterTestPath])` with device serial via `-d` flag
  - Capture stdout/stderr via utf8.decoder + LineSplitter, log each line
  - Parse exit code: 0 = passed, non-zero = failed
  - Build a single StepResult with action "flutter_test", success based on exit code, durationMs from Stopwatch
  - Return TestResult with that single step
- `dispose()` -- close all BehaviorSubjects

Accept `UiTreeService` as optional DI param too (needed for assertion "element_exists" -- check ui tree for element at index). Constructor signature: `TestRunnerService(this._actionService, this._screenshotService, this._deviceManager, this._logService, {UiTreeService? uiTree})`

**openmob_hub/lib/server/routes/test_routes.dart:**
Follow existing pattern from action_routes.dart (pure function returning Router, receives services as DI params).

```dart
Router testRoutes(TestRunnerService runner) {
  final router = Router();
  // endpoints below
  return router;
}
```

Endpoints:
- `GET /` -- returns all scripts as JSON array: `router.get('/', ...)` -> `jsonEncode(runner.scripts$.value.map((s) => s.toJson()).toList())`
- `GET /results` -- returns all test results: `jsonEncode(runner.results$.value.map((r) => r.toJson()).toList())`
- `GET /results/current` -- returns current running test or null: `jsonEncode(runner.currentRun$.value?.toJson())`
- `POST /` -- create/save a test script from JSON body: parse body as TestScript.fromJson, call runner.addScript, return 201 with script JSON
- `DELETE /<id>` -- remove script by id: runner.removeScript(id), return 200 with `{"deleted": true}`
- `POST /<id>/run` -- execute a test script: call runner.runScript(id), return result JSON. Use try/catch, return 500 on error.
- `POST /flutter-test` -- run a flutter test: parse body `{"path": "test/widget_test.dart", "device_id": "optional"}`, create a temporary TestScript with flutterTestPath set, call runner.runScript, return result
